# 《最终实现功能清单（规则文件版）》 

## 适用范围说明
本清单用于明确系统功能的实现边界，区分核心实现功能、扩展实现功能与设计说明功能，作为系统开发与论文撰写的统一依据。 

## 一、必做功能（100% 必须实现） 

### 1️⃣ 用户与角色管理（RBAC 基础版） 

* 用户登录 / 注销 
* 基本角色划分：
  * 管理员 
  * 教师 
  * 学生 
* 基于角色的功能权限控制 
* JWT 身份认证 
* Spring Security 权限校验 

📌 **规则说明**
* 不做角色继承 
* 不做动态权限配置界面 
* 角色权限可通过代码或初始化数据配置 

### 2️⃣ 设备全生命周期管理（系统核心） 

* 设备信息维护（增 / 删 / 改 / 查） 
* 设备状态管理：
  * 待入库 
  * 已入库 
  * 使用中 
  * 维修中 
  * 报废 
* 状态流转规则校验 
* 设备必须关联到某个实验室，不能是自由文本位置 

📌 **这是论文和系统的“主干”，必须稳定可演示** 

### 3️⃣ 实验室管理与预约（规则校验型） 

* 实验室信息管理 
* 实验室可用状态维护 
* 实验室预约申请 
* 时间冲突检测 
* 固定审批流程：
  * 申请人 → 管理员 
* 预约状态管理（待审 / 通过 / 拒绝） 

📌 **不做**：
* 自动调度算法 
* 智能排程 

### 4️⃣ 设备借用与归还流程 

* 借用申请 
* 管理员审批 
* 借用记录管理 
* 归还确认 
* 逾期状态标记 

📌 **流程必须能完整跑通** 

### 5️⃣ 维修与报废管理（简化版） 

* 维修申请 
* 维修状态记录（仅做状态流转，不做维修人员指派与进度跟踪） 
* 报废申请 
* 报废审批（管理员） 

📌 **不涉及费用统计、不涉及外部维修系统** 

### 6️⃣ 基础统计与提醒（固定场景） 

* 设备使用次数统计 
* 借用 / 逾期统计 
* 简单到期提醒（如预约到期前）：仅页面红点+个人中心列表高亮，不做任何推送机制 

📌 **固定页面即可，不做报表配置** 

### 7️⃣ AI 智能助手（单轮问答 · 重点亮点） 

* 单轮自然语言输入 
* 关键词 / 规则匹配 
* 本地知识库优先 
* 未命中时调用大模型 API 
* 支持典型场景：
  * 设备位置查询 
  * 借用流程说明 
  * 预约规则说明 

📌 **明确规则**
* 不维护多轮上下文 
* 不训练模型 
* 不允许模型直接生成业务数据 

## 二、选做功能（时间允许可实现，不影响毕业） 

### 1️⃣ 实验课程信息管理（非自动排课） 

* 课程基本信息维护 
* 课程与实验室、设备关联展示 
* 人工录入课程安排（仅做展示，不提供编辑功能） 

📌 **不实现自动排课、不实现冲突优化** 

### 2️⃣ 借用 / 预约提醒增强 

* 预约前提醒 
* 逾期提醒消息 

📌 可用：
* 系统消息 
* 页面提示 
  不强制短信 / 邮件 

### 3️⃣ AI 知识库维护界面 

* FAQ 配置 
* 关键词维护 

📌 可后台简单页面实现

## 三、重要边界补充声明（必须阅读）

1. 本系统**不追求**大而全，优先保证核心主流程稳定可演示
2. 所有涉及「审批」的功能均为**固定两级**（申请人→管理员），不存在多级、多角色、自定义流程
3. 前端页面总数**严格控制**在 25 个以内（包括登录、个人中心等）
4. 演示数据规模：实验室≤5个，设备≤50台，注册用户≤100人
5. AI模块**严禁**让大模型直接生成/修改任何业务数据，仅用于咨询、解释、指引
6. 任何选做功能在**不影响必做功能完成度与稳定性**的前提下才考虑实现
